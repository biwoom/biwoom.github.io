# OL HOME

OL HOME은 OL 프로젝트의 공개 웹 홈입니다. OL은 불교 문헌, 이야기, 시각 자료, 작업 기록, 지식 엔티티를 디지털 환경에 맞게 정리하고 공개하는 불교 콘텐츠 프로젝트입니다.

OL은 백과사전식 나열보다 문맥과 개념이 서로 이어지는 학습 환경을 지향합니다. 이름 "올"은 실의 올, 위로 오름, 전체를 뜻하는 "올"이라는 세 층위를 함께 담고 있습니다. 경전의 날실과 논서의 씨실처럼, 개별 문장과 개념을 하나씩 엮어 사용자가 불교의 가르침을 자신의 삶과 연결해 이해하도록 돕는 것이 목표입니다.

공식 사이트:

```txt
https://biwoom.github.io/
```

저장소:

```txt
https://github.com/biwoom/biwoom.github.io
```

## Brand

**브랜드명**: 올 / OL  
**태그라인**: 실처럼 이어진 지혜  
**영문 태그라인**: Thread by thread, truth unfolds  
**슬로건**: 지혜의 올을 짜다

OL의 핵심 질문은 단순합니다.

> 사람들이 이 경험을 통해 정말 더 자유로워지는가?

OL은 정보 전달만을 목표로 하지 않습니다. 불교 콘텐츠를 통해 고통의 감소, 자비와 이타심, 존재에 대한 통찰, 명상과 수행의 가능성을 나누는 것을 목표로 합니다.

## Philosophy

OL은 다음 세 가지 가치를 중심에 둡니다.

- **연결**: 고립된 텍스트와 개념을 살아있는 연결망으로 엮습니다.
- **깊이**: 빠른 소비보다 천천히 읽고 사유하는 경험을 우선합니다.
- **여정**: 사용자가 자신만의 질문과 관심을 따라 불교를 탐색하도록 돕습니다.

OL은 문사수의 흐름을 디지털 콘텐츠 환경 안에서 구현하려 합니다.

- **문(聞)**: 불교 문헌과 이야기를 읽고 듣습니다.
- **사(思)**: 질문하고, 사유하고, 개념을 서로 연결합니다.
- **수(修)**: 명상하고 실천하며 삶 속에서 체화합니다.

## Product System

현재 OL HOME은 다음 제품 영역을 중심으로 운영됩니다.

- `HOME`: 프로젝트 입구와 최신 업데이트
- `TEXT`: 경론 중심의 번역, 주석, 연구 문서
- `STORY`: 이야기 불교와 대중적 불교 콘텐츠
- `DESIGN`: 인포그래픽, 삽화, 시각 자료
- `NET`: 인물, 장소, 개념 이름카드를 중심으로 STORY, TEXT, DESIGN, ENTITY 상세를 잇는 탐색 메뉴
- `BLOG`: 콘텐츠 개발일지와 일상 명상 기록
- `ATLAS`: 정적 HTML 도구와 아카이브

`ENTITY`는 공개 상단 메뉴가 아니라 개별 상세 문서와 내부 데이터 구조로 유지됩니다.

`BOOK`과 `WORKS` 라우트는 과거 구조와 호환을 위해 남아 있지만, 신규 운영의 중심은 `TEXT`, `STORY`, `DESIGN`, `NET`, `BLOG`입니다.

## Content Direction

### TEXT: 경론 중심의 번역과 주석

TEXT는 불교 문헌을 번역하고 주석하며, 깊이 있는 학습과 연구의 기반을 만드는 영역입니다.

주요 제작 방향:

- 개별 경전과 논서 번역물
- 삼론종, 중관학파 계열 문헌 번역
- 정토종 계열 문헌 번역
- 금강경, 원각경, 유마경 등 개별 경론 번역과 주석
- 원효 스님 저서 번역과 주석
- 김성철 교수님 강의록과 연구논문
- 중앙승가대 중관학 강의록
- 불광 인도불교사 강의록
- 보리도차제론 강의록
- 선어록 번역과 해석
- 수행연구

### STORY: 이야기 불교, 대중적 불교 콘텐츠

STORY는 불교의 인물, 가르침, 설화, 수행의 장면을 이야기 형식으로 재구성하는 영역입니다. 전문 지식이 없는 독자도 자연스럽게 읽고 이해할 수 있는 대중적 불교 콘텐츠를 지향합니다.

주요 제작 방향:

- 이야기 불교
- 붓다 스토리: 부처님 일대기
- 담마빠다 이야기
- 고승전 등 전기류와 선사들의 일대기
- 자따까
- 불교 명상 동화책
- 수필집
- 선시
- 불교 경론 게송: 부처님과 제자들의 노래

### DESIGN: 시각적 불교 콘텐츠

DESIGN은 불교 개념과 이야기를 시각적으로 이해할 수 있도록 돕는 영역입니다. 복잡한 교리와 문헌 구조를 더 직관적으로 파악할 수 있게 만드는 것을 목표로 합니다.

주요 제작 방향:

- 불교 콘텐츠 인포그래픽
- 이야기 불교와 불교 명상 동화책 삽화
- 불교 인물, 지역, 건물, 의상 관련 스타일시트

### BLOG: 불교 콘텐츠 개발일지와 일상 명상일지

BLOG는 콘텐츠가 만들어지는 과정과 일상 수행의 기록을 남기는 영역입니다. 완성된 결과물뿐 아니라 제작 과정의 판단, 시행착오, 요약 설명도 함께 보존합니다.

주요 제작 방향:

- TEXT, STORY, DESIGN 콘텐츠 개발 과정 기록
- 각 불교 콘텐츠의 요약과 해설
- 일상 수행과 불교 잠언

## Audience

OL은 우선 일반 대중을 바라봅니다.

- 불교를 처음 접하는 사람
- 삶의 의미를 찾고 있는 사람
- 마음의 고통을 줄이고 싶은 사람
- 수행에 관심은 있으나 어렵게 느끼는 사람
- 깊은 이야기를 원하지만 지루한 설명은 원하지 않는 사람

따라서 OL의 콘텐츠는 쉽고, 명료하며, 아름답고, 집중 가능해야 합니다.

## Design Principles

OL의 디자인은 사용자가 디자인 자체보다 콘텐츠에 집중하도록 돕는 것을 목표로 합니다.

- **모던**: 무채색 기반, 정확한 보더, 정제된 레이아웃
- **심플**: 장식보다 가독성과 흐름 우선
- **세련됨**: 정밀한 간격, 일관된 radius, 안정적인 타이포그래피
- **질리지 않음**: 강한 취향의 장식과 과도한 컬러 사용 지양

OL은 사용자를 오래 붙잡아 두는 기능보다, 사용자가 읽고 사유한 뒤 자신의 삶으로 더 잘 돌아가게 하는 경험을 우선합니다.

## Content Model

Content collections are defined in `src/content.config.ts`.

Current collection groups:

- `blog`: project logs, essays, and meditation notes
- `text`: translated or annotated Buddhist documents
- `story`: narrative series and chapters
- `design`: visual works, infographics, and design assets
- `entities`: ontology entries for persons, places, concepts, texts, events, practices, and schools
- `ontology`: relation and entity type documentation
- `pages`: permanent project pages
- `ai`: reserved for AI workflow records

Design and story source assets are stored next to their content source under `assets/`. During development and build, `scripts/sync-content-assets.mjs` syncs those assets into `public/generated/` by copying changed files and removing stale generated files.

All collections use `tags: string[]` as the single tag field. Tags must use slash-form `prefix/value`, for example `개념/연기`, `종류/인포그래픽`, or `kind/blog`.

## Commands

```sh
npm install
npm run dev
npm run check
npm run build
npm run preview
```

Command behavior:

- `npm run dev`: syncs content assets, then starts Astro dev server
- `npm run check`: validates content frontmatter, tag syntax, asset existence, and forbidden junk files
- `npm run build`: runs the content check, syncs assets, builds the static site, then creates the Pagefind index
- `npm run preview`: previews the built site locally

The project requires Node.js `>=22.12.0`.

## GitHub Pages

This repository is intended to be a GitHub user page repository. The repository name should be:

```txt
biwoom.github.io
```

Because it is served from the root GitHub Pages URL, `astro.config.mjs` sets:

```js
site: 'https://biwoom.github.io'
```

There is no Astro `base` path. Internal links and assets are generated from `/`.

Deployment is handled by:

```txt
.github/workflows/deploy.yml
```

The GitHub repository should use:

```txt
Settings -> Pages -> Source -> GitHub Actions
```

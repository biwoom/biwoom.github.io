## 0. v0.3 업데이트 핵심

v0.3은 v0.2의 기본 원칙을 유지하면서, OL HOME 공개 운영 기준을 반영한다.

이번 버전의 핵심 변경 사항은 다음과 같다.

```txt
1. STORY 서사본을 2단계로 구분한다.
   - 1차: 원전서사본
   - 2차: 대중서사본

2. 원전서사본은 원전의 사건·사유·비유·교리 구조를 충실히 옮기는 기준본으로 삼는다.

3. 대중서사본은 원전서사본을 바탕으로 비불자 일반 독자가 감동과 이해를 얻을 수 있도록 재구성하는 공개 서사본으로 삼는다.

4. 대중서사본에서도 원전의 핵심 의미·사건·교리 방향은 바꾸지 않는다.

5. 비불자 독자를 위한 짧은 설명, 용어 풀이, 감응적 문장, 절제된 심리 묘사는 허용한다.

6. 원전서사본은 OL HOME에 공개하지 않는다.

7. OL HOME의 STORY 문서 YAML은 공개용 대중서사본을 기준으로 작성한다.

8. 원고 검토·비평·수정 제안에서는 기본적으로 상호 존칭을 사용한다.
   단, 사용자가 학술적 검토를 요청한 경우에는 ‘~이다’식 단정형 문체를 사용할 수 있다.
```

## 1. 문서의 목적

이 문서는 OL PROJECT에서 제작하는 모든 STORY 콘텐츠의 원전 기반 서사 재구성 규칙을 정리한 문서다.

OL STORY는 불교 원전을 단순히 직역하거나 요약하는 콘텐츠가 아니다. 또한 원전의 사건과 가르침을 임의로 변형하여 새로운 이야기를 만드는 창작소설도 아니다.

OL STORY는 불교 원전과 전승에 근거한 사건, 인물, 장소, 가르침을 현대 독자가 읽을 수 있는 한국어 서사로 다시 구성하는 작업이다.

따라서 OL STORY의 기본 성격은 다음과 같다.

```txt
원전 번역
+ 의미 중심 의역
+ 장면 중심 서사화
+ 독자 친화적 문체
+ 원전 근거 보존
= 원전 근거형 서사 재구성
```

이 문서의 핵심 목적은 다음이다.

```txt
1. 원전의 권위와 의미를 보존한다.
2. 현대 독자가 읽을 수 있는 서사로 재구성한다.
3. 특정 불교 전통의 우열을 판단하지 않는다.
4. STORY와 TEXT의 역할을 구분한다.
5. 원전 출처와 창작 범위를 분명히 한다.
6. 원전서사본과 대중서사본의 역할을 구분한다.
7. 향후 YAML, TEXT, ENTITY, DESIGN, NET 연결의 기준을 마련한다.
```

## 2. 적용 범위

이 규칙은 붓다스토리에만 적용되는 규칙이 아니다.

이 규칙은 향후 OL PROJECT에서 제작할 모든 STORY 작품에 적용된다.

적용 대상 예시:

```txt
붓다스토리
담마빠다 이야기
자따까 이야기
고승전과 선사 전기
불교 설화
불교 명상동화
불교 경전 서사화 콘텐츠
불교 인물 중심 서사 콘텐츠
불교 수행 이야기
불교 교리를 장면화한 서사 콘텐츠
```

STORY 콘텐츠가 특정 원전, 전승, 설화, 주석서, 전기 문헌, 불전문학에 근거하여 작성되는 경우 이 규칙을 따른다.

## 3. 기본 정의

### 3.1 원전

이 문서에서 말하는 원전은 특정 전통의 경전만을 뜻하지 않는다.

OL PROJECT에서 원전은 다음을 모두 포함한다.

```txt
빨리어 경전
산스크리트 불전
한역 경전
티베트역 경전
율장 문헌
아비달마 문헌
대승경전
밀교 문헌
주석서
복주서
불전문학
전기류
고승전
전승 설화
불교사 문헌
```

각 문헌은 자신이 속한 전승과 문맥 안에서 고유한 의미와 가치를 가진다.

### 3.2 원전 근거형 서사 재구성

원전 근거형 서사 재구성이란 다음을 뜻한다.

```txt
원전에 근거한 사건과 의미를
현대 독자가 읽을 수 있는 장면과 문장으로
새롭게 배열하고 표현하되,
원전에 없는 핵심 사실·관계·교리 결론을 임의로 만들지 않는 작업
```

이 작업에서 창작은 허용된다.

다만 창작은 다음 영역에서 이루어진다.

```txt
문장
호흡
장면 배치
단락 구성
감정의 여백
독자 친화적 설명
일상어 표현
```

반대로 다음 영역에서는 창작을 제한한다.

```txt
사건의 핵심 내용
인물 관계
장소 정보
교리 명제
전승의 출처
인물의 결정적 동기
원전 간 차이
```

### 3.3 원전서사본

원전서사본은 1차 재구성본이다.

원전서사본은 원문을 그대로 직역한 번역문은 아니지만, 원전의 사건 순서, 사유 구조, 비유, 독백, 교리적 방향을 최대한 충실하게 보존한 서사 기준본이다.

원전서사본의 목적은 다음과 같다.

```txt
1. 원전의 사건 흐름을 보존한다.
2. 원전의 사유와 비유를 가능한 한 살린다.
3. 원전의 교리적 의미를 훼손하지 않는다.
4. 이후 대중서사본을 만들기 위한 기준본이 된다.
5. 원전 대조 검토와 창작 허용 범위 판단의 근거가 된다.
```

원전서사본은 다음과 같은 성격을 가진다.

```txt
직역보다 읽기 쉽다.
대중서사본보다 원전에 가깝다.
교리 설명보다 원전의 사유 흐름을 보존한다.
감동보다 충실성을 우선한다.
그러나 문장은 현대 한국어 서사로 구성한다.
```

### 3.4 대중서사본

대중서사본은 2차 재구성본이다.

대중서사본은 원전서사본을 바탕으로, 불교를 잘 모르는 비불자 일반 독자가 이야기의 의미를 자기 삶과 연결하여 읽을 수 있도록 문장, 장면, 감정선, 호흡을 다시 구성한 서사본이다.

대중서사본의 목적은 다음과 같다.

```txt
1. 비불자 일반 독자가 쉽게 읽게 한다.
2. 원전의 의미가 삶의 질문으로 다가가게 한다.
3. 불교 전문용어의 장벽을 낮춘다.
4. 원전의 교리 방향을 훼손하지 않으면서 감응성을 높인다.
5. STORY 공개본 또는 일반 독자용 본문으로 활용할 수 있게 한다.
```

대중서사본은 다음과 같은 성격을 가진다.

```txt
원전서사본보다 문장이 부드럽다.
반복되는 비유는 의미를 보존하는 선에서 압축할 수 있다.
독자의 공감을 돕는 짧은 내면 문장을 보탤 수 있다.
전문용어는 일상어와 함께 풀어 쓴다.
교훈을 직접 주입하지 않고 장면과 질문으로 느끼게 한다.
```

### 3.5 원전서사본과 대중서사본의 관계

원전서사본과 대중서사본은 우열 관계가 아니다.

두 문서는 역할이 다르다.

```txt
원전서사본 = 근거와 기준을 세우는 본문
대중서사본 = 독자에게 다가가는 본문
```

두 문서의 관계는 다음과 같다.

```txt
원전
  ↓
원전서사본
  ↓
대중서사본
  ↓
TEXT 톺아보기 / ENTITY / DESIGN / NET 확장
```

대중서사본은 반드시 원전서사본을 기준으로 작성한다.

대중서사본에서 감동은 더할 수 있지만, 의미를 바꾸지는 않는다.
문장은 부드럽게 할 수 있지만, 길의 방향은 원전이 가리키는 곳을 따라야 한다.

## 4. 가장 중요한 원칙

OL STORY의 핵심 원칙은 다음 한 문장으로 정리한다.

```txt
사실은 보수적으로,
표현은 창조적으로.
```

다시 말해 다음과 같다.

```txt
사건·인물·관계·장소·교리 명제는 원전에 근거한다.
장면·문장·호흡·감정 전달은 현대 독자를 위해 창조적으로 구성한다.
```

또는 이렇게 표현할 수 있다.

```txt
창작은 표현의 방식에서 이루어지고,
진실성은 원전의 근거 위에 머문다.
```

v0.3에서 유지·보완하는 실무 원칙은 다음과 같다.

```txt
원전서사본은 원전의 결을 지킨다.
대중서사본은 독자의 마음에 닿게 한다.
두 본문 모두 원전의 방향을 바꾸지 않는다.
```

## 5. 불교 원전 평등 원칙

OL PROJECT는 모든 불교 전통의 원전을 동등한 존중의 대상으로 대한다.

OL PROJECT는 특정 전통의 경전만을 불교 원전의 기준으로 삼지 않는다.

다음 원칙을 따른다.

```txt
빨리어 전통의 경전은 고유한 가치가 있다.
한역 경전은 고유한 가치가 있다.
산스크리트 불전은 고유한 가치가 있다.
티베트역 문헌은 고유한 가치가 있다.
대승경전은 고유한 가치가 있다.
주석서와 불전문학도 전승사적·서사적·사상적 가치가 있다.
```

따라서 OL STORY는 원전을 다룰 때 다음 태도를 취한다.

```txt
전승별 차이는 분명히 한다.
그러나 전승 간 우열은 판단하지 않는다.
```

### 5.1 중심 원전과 가치 우열의 구분

하나의 STORY 작품은 제작상 특정 원전을 중심으로 삼을 수 있다.

예를 들어 붓다스토리는 빨리어 전통의 『마하붓다왐사』를 주요 원전으로 삼을 수 있다. 그러나 이것은 제작상의 기준일 뿐, 『마하붓다왐사』가 다른 불교 전기 문헌보다 우월하다는 뜻이 아니다.

마찬가지로 『불본행집경』, 『방광대장엄경』, 『불소행찬』, 『니다나까타』, 율장 전승, 한역 아함, 대승경전, 주석서 전승도 각각 자신의 문맥과 목적 안에서 동등하게 존중된다.

정리하면 다음과 같다.

```txt
중심 원전 = 해당 회차를 구성하는 제작상의 기준 문헌
참고 원전 = 같은 사건이나 인물을 이해하는 데 참고한 문헌
관련 전승 = 같은 주제를 다르게 전하는 전승
```

중심 원전은 우월한 원전이 아니다.
참고 원전은 부차적 가치의 원전이 아니다.
관련 전승은 열등하거나 후대적이라는 이유로 낮게 취급하지 않는다.

### 5.2 전승 구분 원칙

OL STORY는 서로 다른 전승의 내용을 섞을 수 있다.

다만 독자가 혼동하지 않도록 다음을 지킨다.

```txt
1. 어떤 전승을 중심으로 삼았는지 밝힌다.
2. 어떤 전승을 참고했는지 밝힌다.
3. 전승 간 차이가 중요한 경우 TEXT 톺아보기에서 설명한다.
4. 한 전승의 내용을 다른 전승의 내용인 것처럼 표시하지 않는다.
5. 전승 차이를 우열이나 진위 판정으로 단순화하지 않는다.
```

### 5.3 왜곡에 대한 기준

다음은 왜곡이 아니다.

```txt
대승적 요소를 대승 전승에 근거해 제시하는 것
주석서적 요소를 주석서 전승에 근거해 제시하는 것
불전문학적 장면을 불전문학 전승에 근거해 제시하는 것
한역 경전의 서사를 한역 경전 전승에 근거해 제시하는 것
```

문제가 되는 것은 특정 전승의 내용을 다른 전승의 내용으로 오해하게 만드는 경우다.

예를 들어 다음은 피해야 한다.

```txt
한역 경전에 근거한 장면을 빨리어 니까야의 내용처럼 표시하는 것
주석서 전승에 근거한 해석을 초기 경전 본문의 직접 내용처럼 표시하는 것
대승경전의 표현을 니까야 원문 발화처럼 제시하는 것
불전문학의 장면을 모든 전승이 공통으로 전하는 역사적 사실처럼 단정하는 것
```

그러나 이는 해당 전승의 가치가 낮다는 뜻이 아니다.
오직 출처와 문맥을 정확히 구분하기 위한 규칙이다.

## 6. 독자 설정

OL STORY의 기본 독자는 다음과 같이 설정한다.

```txt
불교에 호감은 있으나 전통적 불자는 아닐 수 있는 사람
불교전문용어를 잘 모르는 사람
고등학교 이상의 일반 상식을 가진 사람
삶의 의미와 고통의 문제에 관심 있는 사람
깊은 이야기를 원하지만 학술적 설명에는 익숙하지 않은 사람
```

따라서 STORY 본문은 다음을 지향한다.

```txt
쉽게 읽힌다.
그러나 가볍지 않다.

전문용어를 줄인다.
그러나 불교적 의미를 희석하지 않는다.

문장은 간결하다.
그러나 사유의 깊이는 유지한다.

이야기로 읽힌다.
그러나 원전 근거를 벗어나지 않는다.
```

### 6.1 원전서사본의 독자

원전서사본의 1차 독자는 작업자와 검토자다.

```txt
원전 대조가 필요한 사람
STORY 초안을 검토하는 사람
원전과 대중서사본 사이의 차이를 확인하려는 사람
TEXT 톺아보기 주제를 추출하려는 사람
```

원전서사본도 일반 독자가 읽을 수 있어야 하지만, 우선순위는 원전 충실성에 둔다.

### 6.2 대중서사본의 독자

대중서사본의 1차 독자는 비불자 일반 독자다.

```txt
불교 용어에 익숙하지 않은 사람
불교 이야기를 처음 접하는 사람
삶의 괴로움과 의미를 묻는 사람
학술 설명보다 이야기를 통해 이해하고 싶은 사람
```

대중서사본은 독자가 다음 질문을 자연스럽게 갖도록 돕는다.

```txt
이 이야기가 내 삶과 어떤 관계가 있는가
나는 무엇을 붙잡고 있는가
나는 무엇을 내려놓을 수 있는가
나는 어떤 길을 찾고 있는가
```

## 7. 언어와 문체 원칙

### 7.1 일상어 우선

STORY 본문에서는 한국어 일상어를 우선 사용한다.

불교전문용어는 가능한 한 쉬운 말로 풀어서 표현한다.

예시:

```txt
무상
→ 모든 것은 머물지 않는다.
→ 붙잡아도 삶은 흘러간다.
→ 가진 것은 남아도 가진 사람은 남지 않는다.

고
→ 마음대로 붙잡을 수 없기에 생기는 괴로움
→ 원하는 대로 되지 않는 삶의 아픔

출리
→ 이 삶의 방식으로는 충분하지 않다는 깊은 자각
→ 반복되는 괴로움에서 벗어나고 싶은 마음

보리심
→ 나만이 아니라 모든 존재가 괴로움에서 벗어나기를 바라는 마음
```

### 7.2 전문용어 단계적 노출

전문용어를 완전히 배제하지는 않는다.

필요한 경우 다음 순서로 제시한다.

```txt
1. 먼저 일상어로 의미를 풀어 쓴다.
2. 그 뒤 필요한 경우 전문용어를 짧게 소개한다.
3. 자세한 교리 설명은 TEXT 톺아보기로 분리한다.
```

예시:

```txt
수메다는 보았다.
가진 것은 남아도, 가진 사람은 남지 않는다는 사실을.
불교에서는 이런 사실을 ‘무상’이라고 부른다.
```

### 7.3 문체

STORY 문체는 다음을 지향한다.

```txt
짧은 문장
명확한 장면
절제된 감정
조용한 여운
일상어 중심
시적이되 과장되지 않은 표현
설명보다 장면 중심
교훈보다 사유 중심
```

피해야 할 문체는 다음이다.

```txt
교조적 문체
강의록식 문체
논문식 문체
지나치게 감상적인 문체
현대 자기계발식 문체
과도한 판타지 소설식 문체
원전보다 작가의 해석이 앞서는 문체
```

### 7.4 원전서사본 문체

원전서사본은 다음 문체를 따른다.

```txt
원전의 사유 순서를 보존한다.
비유와 독백을 가능한 한 생략하지 않는다.
문장은 현대 한국어로 풀어 쓰되, 의미를 과도하게 바꾸지 않는다.
원전의 반복은 필요한 경우 유지한다.
설명은 짧게 넣고, 해설은 TEXT 후보로 분리한다.
```

### 7.5 대중서사본 문체

대중서사본은 다음 문체를 따른다.

```txt
문장 호흡을 부드럽게 한다.
독자가 자기 삶과 연결할 수 있는 질문을 강화한다.
반복되는 표현은 의미를 보존하는 선에서 압축할 수 있다.
불교 전문용어는 일상어 풀이와 함께 제시한다.
교리 결론을 강요하지 않고 장면과 여운으로 느끼게 한다.
```

대중서사본에서 허용 가능한 문장 예시:

```txt
수메다는 처음으로 알았다.
가진 것이 많다는 사실이
반드시 자유롭다는 뜻은 아니라는 것을.
```

이 문장은 원전에 그대로 있지 않더라도, 재산과 죽음을 사유하는 원전 흐름에서 추론 가능하므로 허용될 수 있다.

대중서사본에서 주의해야 할 문장 예시:

```txt
수메다는 세상의 불평등을 해결하기 위해 자신의 재산을 나누기로 결심했다.
```

이 문장은 현대 사회윤리를 원전 인물의 결정적 동기로 투사할 위험이 있으므로 피한다.

## 8. 창작 허용 범위

OL STORY는 창작을 금지하지 않는다.

다만 창작의 범위를 분명히 한다.

### 8.1 모든 STORY에서 허용되는 창작

다음은 허용된다.

```txt
문장 재구성
단락 조정
장면 전환
묘사 추가
호흡 조절
독자 친화적 비유
일상어 풀이
원전 사건의 장면화
추론 가능한 범위의 내면 묘사
원전 근거가 있는 짧은 대화 구성
```

### 8.2 원전서사본에서 허용되는 창작

원전서사본에서는 다음 정도의 창작을 허용한다.

```txt
원전 문장의 자연스러운 한국어화
원전의 압축된 논리를 독자가 이해할 수 있도록 풀어 쓰기
원전 각주나 설명을 본문 안에 짧게 녹이기
장면 전환을 위한 최소한의 문장 추가
원전 행위에서 직접 추론 가능한 내면 묘사
비불자 이해를 위한 짧은 용어 설명
```

예시:

```txt
삭카는 보살과 훌륭한 수행자를 돕는 존재로 전해지는 하늘 신들의 왕이다.
위쑤깜마는 신들의 뛰어난 장인이자 건축가로 전해진다.
```

이런 설명은 원전 각주의 내용을 독자가 이해하기 쉽게 본문에 녹인 것이므로 허용된다.

### 8.3 대중서사본에서 허용되는 창작

대중서사본에서는 원전서사본보다 넓은 표현 재구성을 허용한다.

다음은 허용된다.

```txt
문장 순서 일부 재배열
반복되는 비유의 압축
감응을 위한 짧은 내면 문장 추가
장면의 분위기 묘사 강화
침묵, 거리, 빛, 몸짓 등의 절제된 묘사
현대 독자가 이해할 수 있는 질문형 문장
원전 교리의 쉬운 풀이
```

다만 다음 조건을 지켜야 한다.

```txt
원전의 핵심 사건을 바꾸지 않는다.
인물의 결정적 동기를 새로 만들지 않는다.
교리적 방향을 바꾸지 않는다.
전승 출처를 흐리지 않는다.
현대적 메시지를 원전의 직접 가르침처럼 제시하지 않는다.
```

대중서사본의 핵심 원칙:

```txt
감동은 더할 수 있지만, 의미를 바꾸지는 않는다.
문장은 부드럽게 할 수 있지만, 길의 방향은 원전이 가리키는 곳을 따른다.
```

### 8.4 제한되는 창작

다음은 제한된다.

```txt
원전에 없는 사건 추가
원전에 없는 인물 관계 추가
원전에 없는 결정적 동기 부여
원전에 없는 교리 결론 삽입
원전에 없는 갈등 구조 창작
원전에 없는 인물 성격의 과도한 설정
현대적 가치관을 고대 인물의 발화로 투사
특정 전승을 다른 전승의 내용처럼 제시
```

### 8.5 금지되는 창작

다음은 금지한다.

```txt
원전의 핵심 사건을 반대로 바꾸는 것
인물의 수행상태를 임의로 바꾸는 것
전승에 없는 가족관계나 사제관계를 만드는 것
역사적·전승적 인물을 허구 인물처럼 변형하는 것
원전의 교리적 의미를 반대 방향으로 왜곡하는 것
출처 없는 현대적 메시지를 불교 원전의 가르침처럼 제시하는 것
특정 전승을 낮추거나 희화화하는 것
```

## 9. 대화 구성 원칙

원전에 없는 대화를 추가할 수 있다.

그러나 다음 조건을 모두 만족해야 한다.

```txt
1. 대화의 핵심 내용이 원전 사건에 근거해야 한다.
2. 그 인물이 해당 시점에서 알 수 있는 내용이어야 한다.
3. 해당 인물의 수행상태와 전승 문맥에 맞아야 한다.
4. 특정 전승의 교리 표현을 다른 전승의 직접 발화처럼 제시하지 않아야 한다.
5. 대화는 사건을 왜곡하지 않고 독자의 이해를 돕는 수준이어야 한다.
```

### 9.1 허용 가능한 대화

원전에 어떤 인물이 출가를 결심했다는 내용이 있을 경우, 다음과 같은 내면 독백은 가능하다.

```txt
“이 삶이 전부라면, 왜 마음은 여전히 답을 찾고 있는가.”
```

원전에 어떤 인물이 붓다에게 질문했다는 내용이 있을 경우, 그 질문의 핵심을 현대어로 재구성할 수 있다.

```txt
“어떻게 살아야 이 괴로움에서 벗어날 수 있습니까?”
```

### 9.2 피해야 할 대화

다음과 같은 대화는 피한다.

```txt
원전에 없는 장황한 철학 토론
후대 교리 체계를 과거 인물의 직접 발화로 삽입
현대 심리학적 진단을 고대 인물의 말로 표현
원전 근거가 없는 연애, 가족 갈등, 정치적 음모 대화
작가의 현대적 메시지를 인물의 입에 넣는 것
```

### 9.3 원전서사본과 대중서사본의 대화 차이

원전서사본의 대화는 원전 발화에 더 가깝게 유지한다.

대중서사본의 대화는 의미를 보존하는 선에서 더 자연스럽게 다듬을 수 있다.

다만 대중서사본에서도 다음은 금지한다.

```txt
원전 인물이 알 수 없는 후대 교리를 말하게 하는 것
원전 인물에게 현대적 자기계발 메시지를 말하게 하는 것
인물의 관계를 새로 설정하는 대화를 만드는 것
```

## 10. 심리 묘사 원칙

인물의 내면 묘사는 가능하다.

그러나 원전의 행위, 결정, 상황에서 합리적으로 추론 가능한 범위 안에서만 허용한다.

### 10.1 허용

```txt
부모의 죽음을 본 인물이 삶의 덧없음을 느끼는 장면
병든 사람을 본 인물이 두려움과 질문을 느끼는 장면
늙은 사람을 본 인물이 젊음의 불안정성을 느끼는 장면
수행자가 깨달음을 찾기 위해 고요히 결심하는 장면
재산을 보시하는 인물이 소유의 무게를 내려놓는 장면
출가자가 수행복을 입으며 새 삶의 결의를 느끼는 장면
```

### 10.2 제한

```txt
원전에 없는 트라우마 설정
원전에 없는 원망과 분노 설정
원전에 없는 열등감 설정
원전에 없는 사랑과 질투 설정
원전에 없는 정치적 야망 설정
원전에 없는 가족 갈등 설정
```

인물의 내면은 독자의 몰입을 돕기 위한 장치이지, 원전에 없는 새 인물을 창조하기 위한 장치가 아니다.

### 10.3 대중서사본의 심리 묘사 기준

대중서사본에서는 독자의 감응을 위해 심리 묘사를 더 부드럽게 확장할 수 있다.

예시:

```txt
그가 비운 것은 창고만이 아니었다.
그는 소유의 무게를 비웠다.
```

이 문장은 원전의 대보시와 출가 흐름에서 합리적으로 추론 가능한 심리적 의미를 표현한 것이므로 허용된다.

그러나 다음과 같은 문장은 피한다.

```txt
수메다는 가난한 사람들의 불평등한 삶을 보고 사회를 바꾸겠다고 결심했다.
```

이 문장은 원전의 보시를 현대적 사회개혁 동기로 바꾸는 효과가 있으므로 피한다.

## 11. 불교 용어와 고유명사 처리 원칙

### 11.1 고유명사

불교 고유명사는 원어 음역을 기본으로 하되, 첫 등장 시 독자 이해를 돕는 짧은 설명을 붙일 수 있다.

예시:

```txt
아마라와띠(Amaravatī)
→ 불전 속 도시명. 문자적으로는 “죽지 않는 이들의 도시”, “불멸자들의 거처”라는 뜻을 품은 이름으로 이해할 수 있다.

삭카(Sakka)
→ 불전에서 보살과 훌륭한 수행자를 돕는 존재로 자주 등장하는 하늘 신들의 왕.

위쑤깜마(Vissukamma)
→ 삭카의 명을 받아 수행처를 마련하는 신들의 장인이자 건축가.
```

STORY 본문에서는 설명을 길게 늘리지 않는다.
상세한 어원, 전승 비교, 문헌별 용례는 TEXT 톺아보기로 분리한다.

### 11.2 수행 용어의 대중적 번역

전문용어는 본문에서 독자가 이해할 수 있도록 풀어 쓴다.

예시:

```txt
fibre-robe
→ 나무껍질과 거친 옷감으로 만든 수행복
→ 문맥에 따라 “수행복”으로 줄일 수 있다.

Five Higher Spiritual Powers / abhiññā
→ 다섯 가지 높은 앎과 뛰어난 능력
→ 문맥에 따라 “다섯 가지 뛰어난 능력”으로 줄일 수 있다.

Eight Attainments / eight mundane jhānas
→ 여덟 단계의 깊은 명상 성취
→ 문맥에 따라 “여덟 단계의 깊은 명상”으로 줄일 수 있다.

Private Buddha / Paccekabuddha
→ 홀로 깨달은 성자
→ 필요 시 TEXT에서 “벽지불”을 설명한다.
```

### 11.3 몸과 출리 표현의 주의점

부정관, 몸의 부정함, 출리, 염오와 관련된 표현은 원전 의미를 보존하되, 현대 독자가 “몸 혐오”로 오해하지 않도록 문맥을 조정할 수 있다.

대중서사본에서는 다음과 같은 보완 문장을 허용한다.

```txt
그는 몸 자체를 미워하려는 것이 아니었다.
그는 몸에 대한 집착을 버리려는 것이었다.
```

이 문장은 원전의 출리 방향을 현대 독자에게 오해 없이 전달하기 위한 설명이므로 허용된다.

다만 다음과 같은 표현은 피한다.

```txt
몸은 악한 것이다.
몸은 버려야 할 죄악이다.
```

이는 불교적 출리와 부정관을 도덕적 혐오로 오해하게 만들 수 있다.

## 12. 붓다 표현 원칙

OL STORY의 본문 서술에서는 “붓다”라는 표현을 기본으로 사용한다.

예외적으로 다음 경우에는 다른 표현을 사용할 수 있다.

```txt
원전 제목
직접 인용문
전승상 고유 표현
인물의 호칭 차이를 설명하는 TEXT 문서
YAML의 aliases 필드
검색용 태그
```

본문 서술의 기본 원칙은 다음과 같다.

```txt
성도 전: 싯다르타, 보살, 수행자 고타마
성도 후: 붓다
문맥상 필요할 때: 고타마 붓다
```

“부처님”, “세존”, “여래” 등의 표현은 원전 문맥이나 직접 인용, 해설 문서, 태그, 별칭에서는 사용할 수 있으나, STORY 본문 기본 서술명은 “붓다”로 통일한다.

### 12.1 성도 전후 시점 구분

성도 이전의 싯다르타나 보살에게 성도 이후의 완성된 깨달음을 이미 아는 사람처럼 말하게 해서는 안 된다.

성도 전에는 다음과 같이 표현한다.

```txt
찾는 사람
묻는 사람
길을 시험하는 사람
괴로움의 끝을 구하는 사람
```

성도 후에는 다음과 같이 표현한다.

```txt
본 사람
깨어난 사람
길을 보인 사람
괴로움의 끝을 가르친 사람
```

## 13. STORY와 TEXT의 분리 원칙

STORY는 이야기의 공간이다.

TEXT는 해설의 공간이다.

따라서 STORY 본문에는 다음을 과도하게 넣지 않는다.

```txt
원전 비교
전승 차이 분석
교리 논쟁
학술적 주석
용어사적 설명
문헌학적 설명
종파별 해석 비교
```

이 내용은 STORY 작성 후 TEXT 톺아보기 후보로 분리하여 제시한다.

### 13.1 STORY가 담당하는 것

```txt
사건
장면
인물
선택
질문
감정의 흐름
독자의 몰입
원전 의미의 서사적 전달
```

### 13.2 TEXT가 담당하는 것

```txt
원전 비교
전승 차이
교리 해설
학술적 설명
현대적 의미 분석
문헌 출처 정리
용어 설명
사상사적 위치
```

### 13.3 STORY 작성 후 제시해야 할 TEXT 톺아보기 후보

각 STORY 회차를 재구성한 뒤, 작성자는 해당 회차에서 TEXT로 분리할 만한 주제를 제시한다.

예시:

```txt
TEXT 톺아보기 후보

1. 수메다 전승은 어떤 문헌들에 나타나는가
2. 수기란 무엇인가
3. 보살도는 어떻게 시작되는가
4. 마하붓다왐사와 불본행집경의 전승 차이
5. 전생 서원 이야기를 현대 독자는 어떻게 읽을 수 있는가
```

이 후보는 즉시 TEXT 문서로 작성하지 않아도 된다.
다만 STORY 본문에서 분리된 해설 주제를 기록하여 이후 콘텐츠 확장에 활용한다.

## 14. 원전 출처 표기 원칙

각 STORY 회차에는 근거 카드를 별도로 붙이지 않는다.

대신 STORY 본문 하단에 원전 출처 문구를 둔다.

### 14.1 원전 출처 문구의 목적

원전 출처 문구는 다음을 독자에게 알려준다.

```txt
이 이야기가 어떤 원전에 근거했는가
어떤 전승을 중심으로 재구성했는가
어떤 문헌을 함께 참고했는가
전승 차이가 있을 수 있음을 어떻게 안내할 것인가
```

### 14.2 원전 출처 문구 기본 형식

```md
## 원전 출처

이 이야기는 [중심 원전명]의 [해당 장·품·절]을 중심으로,
[참고 원전명]과 [관련 전승명]을 함께 참고하여 원전 근거형 서사로 재구성했습니다.

전승에 따라 사건의 세부 묘사와 해석에는 차이가 있을 수 있습니다.
OL STORY는 각 전승의 가치를 우열로 판단하지 않고, 필요한 경우 TEXT 톺아보기에서 전승별 차이를 따로 설명합니다.
```

### 14.3 원전서사본 출처 문구 예시

```md
## 원전 출처

이 원전서사본은 빨리어 전통의 『마하붓다왐사』 중 수메다 보살 전승을 중심으로,
해당 원전의 사건 순서와 사유 구조를 최대한 충실하게 보존하여 재구성했습니다.

문장은 현대 한국어 서사로 다듬었으나,
핵심 사건, 인물 관계, 수행 과정, 교리적 의미는 중심 원전에 근거합니다.
```

### 14.4 대중서사본 출처 문구 예시

```md
## 원전 출처

이 대중서사본은 같은 회차의 원전서사본을 바탕으로,
비불자 일반 독자가 이야기를 쉽게 이해하고 자기 삶의 질문으로 받아들일 수 있도록 다시 재구성했습니다.

원전의 핵심 사건과 교리적 방향은 보존했으며,
문장과 장면의 호흡은 현대 독자를 위해 새롭게 다듬었습니다.
```

### 14.5 붓다스토리 예시

```md
## 원전 출처

이 이야기는 빨리어 전통의 『마하붓다왐사』 중 수메다 보살 전승을 중심으로,
한역 불전인 『불본행집경』 등 관련 불전 전승을 함께 참고하여 원전 근거형 서사로 재구성했습니다.

전승에 따라 사건의 세부 묘사와 해석에는 차이가 있을 수 있습니다.
OL STORY는 각 전승의 가치를 우열로 판단하지 않고, 필요한 경우 TEXT 톺아보기에서 전승별 차이를 따로 설명합니다.
```

## 15. YAML 작성 원칙

원전을 근거로 STORY 서사를 재구성한 후에는 해당 STORY 문서의 YAML 정보를 작성한다.

YAML은 단순한 메타데이터가 아니다.
YAML은 공개 STORY가 OL NET, TEXT, DESIGN, ENTITY와 연결되기 위한 기본 정보다.

### 15.1 STORY YAML 작성 대상

STORY 회차를 작성한 뒤 다음 YAML 정보를 작성한다.

```txt
title
kind
series
seriesSlug
seriesOrder
part
partSlug
partOrder
group
groupSlug
groupOrder
category
chapter
order
version
status
publishedAt
description
primaryEntities
primaryPlaces
tags
tagAliases
sourceTraditions
relatedTextCandidates
license
rightsHolder
lang
published
```

### 15.2 원전서사본 관리와 공개 YAML의 구분

원전서사본은 OL HOME에 공개하지 않는다.

따라서 공개용 `src/content/story/...` 문서에는 원전서사본과 대중서사본을 구분하기 위한 판본 관리 필드를 기본적으로 넣지 않는다.

다음 필드는 공개 사이트 YAML이 아니라 외부 제작·검토 경로의 관리 메타데이터로 다룬다.

```yaml
storyForm: "source-narrative" # 원전서사본
storyForm: "public-narrative" # 대중서사본
sourceNarrativeOf: "원전서사본 문서 id 또는 경로"
```

이 필드는 외부 제작·검토 단계에서 다음 목적을 가진다.

```txt
원전서사본과 대중서사본을 구분한다.
대중서사본이 어떤 원전서사본을 바탕으로 했는지 표시한다.
편집 관리에서 판본 관계를 추적할 수 있게 한다.
```

공개 사이트에 반영할 때는 대중서사본 또는 공개용 STORY 본문만 사용한다.

### 15.3 STORY YAML 예시

```yaml
---
title: "수메다, 새로운 길을 찾다"
kind: "document"

series: "붓다스토리"
seriesSlug: "buddha-story"
seriesOrder: 10

part: "프롤로그"
partSlug: "part-0"
partOrder: 0

group: "먼 과거의 서원"
groupSlug: "ancient-vow"
groupOrder: 10

category: "불교서사"

chapter: 1
order: 1

version: "v0.2.0"
status: "draft"
publishedAt: 2026-06-18

description: "수메다가 삶의 덧없음을 마주하고 사라지지 않는 길을 묻기 시작하는 장면을 원전 근거형 대중서사로 재구성한 이야기입니다."

primaryEntities:
  - "sumedha"

primaryPlaces:
  - "amaravati"

sourceTraditions:
  - title: "Mahābuddhavaṃsa"
    role: "중심 원전"
    note: "수메다 보살 전승"
  - title: "佛本行集經"
    role: "참고 전승"
    note: "관련 붓다 전기 전승"

relatedTextCandidates:
  - "수메다는 누구인가"
  - "수기란 무엇인가"
  - "보살도는 어떻게 시작되는가"
  - "마하붓다왐사와 한역 불전의 수메다 전승 비교"

tags:
  - "형식/불교서사"
  - "작품/붓다스토리"
  - "인물/수메다"
  - "장소/아마라와띠"
  - "주제/무상"
  - "주제/출가"
  - "주제/보살도"
  - "경전/마하붓다왐사"
  - "경전/불본행집경"

tagAliases:
  "인물/수메다": ["Sumedha", "수메다 보살", "수메다 브라만"]
  "장소/아마라와띠": ["Amaravatī", "Amaravati", "아마라바티"]
  "주제/무상": ["anicca", "덧없음", "머물지 않음"]
  "주제/보살도": ["bodhisatta path", "bodhisattva path", "보살의 길"]

license: "CC0"
rightsHolder: "비움"
lang: "ko"
published: true
---
```

### 15.4 YAML 작성 시 주의점

YAML에는 STORY 문서가 실제로 연결할 수 있는 정보만 우선 작성한다.

불확실하거나 아직 제작되지 않은 항목은 무리하게 채우지 않는다.

특히 다음은 주의한다.

```txt
primaryEntities는 실제 Entity id와 연결될 수 있어야 한다.
primaryPlaces는 실제 Place Entity id와 연결될 수 있어야 한다.
sourceTraditions에는 중심 원전과 참고 전승을 구분한다.
relatedTextCandidates는 작성 완료 문서가 아니라 후보 주제일 수 있다.
tags는 사용자 탐색과 필터를 위한 표현으로 작성한다.
원전서사본 판본 관리 필드는 공개 STORY YAML에 넣지 않는다.
```

## 16. ENTITY 작성 원칙

인물, 장소 ENTITY 내용과 YAML 정보는 사용자의 요청이 있을 때 작성한다.

STORY 회차를 작성할 때마다 자동으로 ENTITY 문서를 작성하지 않는다.

### 16.1 기본 원칙

```txt
STORY 작성 = 기본 작업
STORY YAML 작성 = 기본 작업
TEXT 톺아보기 후보 제시 = 기본 작업
ENTITY 작성 = 사용자 요청 시 작업
DESIGN 작성 = 사용자 요청 시 작업
```

### 16.2 ENTITY 작성이 필요한 경우

다음 경우 사용자의 요청에 따라 ENTITY 문서를 작성한다.

```txt
새로운 핵심 인물이 등장한 경우
새로운 핵심 장소가 등장한 경우
인물 관계망 구축이 필요한 경우
NET 검색과 필터를 위해 원본 데이터가 필요한 경우
STORY 인물카드가 필요한 경우
장소 지도 연결이 필요한 경우
```

### 16.3 ENTITY 작성 시 기준

ENTITY 문서를 작성할 때는 STORY 본문보다 더 엄밀하게 원전 정보를 정리한다.

인물 Entity에는 다음 정보가 들어갈 수 있다.

```txt
이름과 별칭
원어명
출신지
거주지
가족관계
스승·제자관계
군신관계
전생 인연
간략 이력
외형적 특징
수행상태
등장 STORY
관련 TEXT
관련 DESIGN
원전별 전승 차이
```

장소 Entity에는 다음 정보가 들어갈 수 있다.

```txt
장소명
원어명
별칭
장소 유형
소속 왕국
현대 위치
좌표
좌표 신뢰도
관련 인물
관련 STORY
관련 TEXT
원전별 전승 차이
```

그러나 이 작업은 STORY 회차 작성의 필수 단계가 아니라, 별도 요청이 있을 때 수행한다.

## 17. 제작 워크플로우

OL STORY의 기본 제작 순서는 다음과 같다.

```txt
1. 대상 원전과 사건 범위 확인
2. 중심 원전과 참고 전승 구분
3. 원전의 핵심 사건 단위 추출
4. 원전의 사유·비유·독백 구조 추출
5. 원전서사본 작성
6. 원전서사본 대조 검토
7. 대중서사본 작성
8. 대중서사본 독자 감응 검토
9. STORY 본문 하단 원전 출처 문구 작성
10. STORY YAML 작성
11. TEXT 톺아보기 후보 제시
12. 사용자 요청 시 ENTITY 작성
13. 사용자 요청 시 DESIGN 자료 작성
```

### 17.1 원전 확인

작업 전 다음을 확인한다.

```txt
어떤 문헌에 근거하는가
어떤 장·품·절에 해당하는가
같은 사건을 다루는 다른 전승이 있는가
전승 간 차이가 중요한가
이 회차에서 중심으로 삼을 사건은 무엇인가
```

### 17.2 사건 단위 추출

원전에서 사건 단위를 추출한다.

예시:

```txt
도시 배경
인물 출신
부모의 죽음
재산 상속
삶의 덧없음에 대한 사유
출가 결심
수행
붓다와의 만남
수기
```

### 17.3 사유·비유·독백 구조 추출

특히 불교 STORY에서는 사건만이 아니라 사유 구조가 중요하다.

다음 요소를 따로 추출한다.

```txt
인물이 무엇을 보았는가
무엇을 괴로움으로 인식했는가
어떤 비유로 이해했는가
어떤 결심에 이르렀는가
그 결심이 어떤 행동으로 이어졌는가
```

수메다 출가 서사의 경우 다음과 같은 요소가 해당한다.

```txt
태어남·늙음·병듦·죽음에 대한 사유
몸의 부정함에 대한 사유
열반의 가능성에 대한 비유
연못·길·의사 비유
몸을 버리는 비유
재산과 죽음에 대한 사유
보시와 출가의 결심
```

### 17.4 원전서사본 작성

원전서사본은 다음 기준으로 작성한다.

```txt
원전 사건 순서 보존
원전 비유 보존
원전 독백 보존
원전 교리 방향 보존
해설은 최소화
비불자 설명은 짧고 자연스럽게 삽입
```

### 17.5 원전서사본 검토

검토 시 다음을 확인한다.

```txt
과도한 생략이 없는가
원전 비유가 사라지지 않았는가
원전 사유의 논리가 유지되었는가
현대적 가치관이 원전의 동기로 바뀌지 않았는가
등장인물의 감정과 관계가 과도하게 추가되지 않았는가
```

### 17.6 대중서사본 작성

대중서사본은 다음 기준으로 작성한다.

```txt
원전서사본을 기준으로 한다.
비불자 독자가 이해할 수 있게 문장을 부드럽게 한다.
사유를 독자의 삶과 연결할 수 있게 한다.
장면의 감정선을 강화한다.
반복을 적절히 압축한다.
전문용어를 일상어와 함께 풀어 쓴다.
```

### 17.7 대중서사본 검토

검토 시 다음을 확인한다.

```txt
감동을 위해 원전 의미를 바꾸지 않았는가
현대 자기계발식 메시지로 흐르지 않았는가
교리적 방향이 원전과 일치하는가
인물의 결정적 동기를 새로 만들지 않았는가
비불자 독자가 읽기 쉬운가
장면과 여운이 살아 있는가
```

## 18. 검토와 피드백 문체 원칙

OL STORY 원고 검토, 원전 대조, 개작 여부 판단, 수정 제안, 기획문서 피드백에서는 기본적으로 상호 존칭을 사용한다.

기본 검토 문체는 다음과 같다.

```txt
~로 보입니다.
~가 적절하겠습니다.
~로 조정하시면 좋겠습니다.
~는 주의가 필요해 보입니다.
~는 허용 가능한 범위로 보입니다.
```

다음과 같은 단정형 문체는 사용자가 학술적 검토를 요청한 경우에 사용한다.

```txt
~이다.
~로 판단된다.
~에 해당한다.
~로 보아야 한다.
```

정리하면 다음과 같다.

```txt
일반 검토·상호 논의 = 존칭과 완곡한 검토 문체
학술적 검토 요청 = 단정형 학술 문체 가능
```

```txt
[ ] 원전 또는 전승 근거가 확인되었는가

[ ] 전승 간 우열 판단을 하지 않았는가

[ ] 붓다 표현 원칙을 지켰는가
[ ] STORY와 TEXT의 역할을 구분했는가
[ ] 본문 하단에 원전 출처 문구를 작성했는가
[ ] STORY YAML을 작성했는가
[ ] TEXT 톺아보기 후보를 제시했는가
```

### 19.2 원전서사본 점검표

```txt
[ ] 원전 사건 순서가 보존되었는가

[ ] 원전 각주의 필요한 정보가 자연스럽게 반영되었는가
```

## 20. 최종 원칙

OL STORY는 원전을 현대 독자가 읽을 수 있는 이야기로 다시 숨 쉬게 하는 작업이다.

그러나 이야기가 살아나기 위해 원전을 넘어서지는 않는다.

OL STORY는 특정 전승을 기준으로 다른 전승을 낮추지 않는다.

전승의 차이는 정직하게 드러내고, 전승의 가치는 동등하게 존중한다.

창작은 표현의 방식에서 이루어진다.

근거는 원전과 전승 위에 머문다.

v0.3의 최종 원칙은 다음과 같다.

```txt
원전서사본은 원전의 길을 잃지 않게 한다.
대중서사본은 그 길이 독자의 마음에 닿게 한다.
```

STORY는 독자가 이야기를 만나게 하고,  
TEXT는 그 의미를 깊게 풀어 주며,  
ENTITY는 인물과 장소의 원본 정보를 축적하고,  
NET은 그 모든 것을 서로 이어진 지식의 그물로 보여 준다.

OL STORY의 목표는 독자를 설득하는 것이 아니다.

독자가 하나의 장면 앞에서 멈추고,  
하나의 질문을 자기 삶으로 가져가게 하는 것이다.

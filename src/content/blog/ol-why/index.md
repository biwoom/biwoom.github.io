OL 프로젝트의 출발점은 기술이 아니었다. 법(法)을 만났을 때의 기쁨, 괴로움의 감소, 자비와 이타심 — 이런 것들을 나누고 싶다는 마음이었다.

## 핵심 질문

> *"사람들이 이 경험을 통해 정말 더 자유로워지는가?"*

이 질문이 OL의 모든 기술적·디자인적·콘텐츠적 결정의 기준점이다. OL은 정보 전달이 아니라, 고통 감소와 삶의 전환을 돕는 경험을 지향한다.

## 브랜드 체계

OL은 하나의 앱이 아니라 역할이 분리된 체계로 구성된다.

- **OL ATLAS** — 단일 HTML 파일로 동작하는 콘텐츠 뷰어 겸 에디터. 설치도 서버도 필요 없다.
- **OL BOOK** — ATLAS 위에서 동작하는 완결된 불교 콘텐츠. 한 파일을 받아 더블클릭하면 끝이다.
- **OL WORKS** — 번역, 주석, 연구 노트를 공개하는 살아 있는 문서 라이브러리.
- **OL DESIGN** — 불교 지식을 시각화하는 인포그래픽과 보조 삽화·스타일시트.

## 문사수(聞思修) 환경

OL은 지식 관리 시스템이 아니라 문사수 환경에 가깝다. 읽고(聞), 사유하고(思), 실천한다(修). 빠른 소비, 과잉 인터랙션, 게임화는 하지 않는다. 대신 깊은 체류, 조용한 탐색, 의미의 여운을 중요하게 여긴다.

## OL이 하지 않는 것들

- 게이미피케이션 (배지, 점수, 연속 사용일)
- 푸시 알림, 추천 알고리즘
- 소셜 로그인, 사용 통계
- 광고, 평가 기능 (별점, 좋아요, 댓글)

이 목록은 기술적 한계가 아니라 의지적 결정이다. 가능하지만 하지 않는다.

## 저장소 구조

도구와 콘텐츠를 분리하는 것이 핵심이다.

```
OL-PROJECT (GitHub Organization)
├── ol-home       → 홈페이지 포털 (Astro + GitHub Pages)
├── ol-atlas      → 앱 엔진 소스코드
├── ol-book       → 배포형 웹북
└── ol-contents   → 공용 시각 자산 라이브러리
```

atlas는 runtime, book은 canon, contents는 assets, home은 gateway. 이 분리가 장기적으로 건강한 구조를 만든다.

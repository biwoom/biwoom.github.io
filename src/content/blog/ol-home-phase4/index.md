오늘 OL HOME에 적용한 작업들을 정리한다. 페이지별 필터와 페이지네이션 통일, 구조적 개선, 그리고 몇 가지 새로운 기능 추가.

## ATLAS 페이지 — 버전 자동 감지

기존에는 `v0.0.6`이 9곳에 하드코딩되어 있었고, 사이드바 다운로드 카드는 `v0.4.2`로 불일치했다.

`public/atlas/latest/`에 `ol-atlas_v{버전}.html` 파일을 넣으면 빌드 시 `fs.readdirSync`로 자동 감지하고, 버전·파일명·파일 크기를 페이지 전체에 반영하도록 수정했다. 새 버전 배포 시 파일만 교체하고 빌드하면 된다.

## BLOG 페이지 — 필터 + 페이지네이션 + 제목 링크

- 카테고리 칩 필터를 하드코딩에서 동적 생성으로 전환
- 사이드바 ARCHIVE 연도 필터 신설 (카테고리와 조합 가능)
- 6개씩 "더 보기" 페이지네이션 추가
- 포스트 제목 클릭으로 상세 페이지 이동

## DESIGN 페이지 — 모바일 CSS 보강 + 다운로드 버튼

600px 이하에서 상세 페이지 제목 크기, 네비게이션 마진, 하단 패딩을 조정했다. 개별 이미지 상세 페이지 메타데이터 영역 하단에 "이미지 다운로드" 버튼을 추가했다.

## BOOK 페이지 — 동적 필터 + 페이지네이션

하드코딩된 필터 칩("전체 12", "경전 4" 등 — 실제 데이터와 불일치)을 실제 `tags` 데이터에서 동적 생성하도록 교체했다. 8개씩 "더 보기" 페이지네이션도 추가.

## WORKS 페이지 — 카테고리 칩 + 태그 드롭다운 + 시리즈 접기

- 시리즈 헤더에 접기/펼치기 토글 추가 (chevron 회전 애니메이션)
- 카테고리 칩 필터 (시리즈 섹션 위에 배치)
- 태그 드롭다운: 카테고리 칩 우측에 "태그" 버튼. 클릭하면 체크박스 리스트 패널이 열리고 복수 선택(AND 필터)이 가능하다. 태그가 많아져도 공간을 차지하지 않는 설계.

## Pages 컬렉션 — 상설 문서

블로그는 시간순 작업 기록이지만, "존재 선언문"이나 "스타일 가이드" 같은 문서는 프로젝트의 정체성을 정의하는 상설 문서다. 이런 문서를 블로그에 넣으면 시간순 목록에 묻힌다.

`src/content/pages/` 컬렉션을 신설하고, 동적 라우트(`/pages/[slug]`)로 렌더링한다. 푸터의 `#` 링크를 실제 페이지 경로로 연결했다.

## 사이트 설정 통합

`src/lib/site.ts`에 사이트 공통 정보(이름, 이메일, GitHub URL 등)를 모았다. 푸터에 하드코딩되어 있던 URL들을 이 설정으로 교체하고, "기여하기"와 "연락처"를 "문의 및 기여"로 합쳤다.

OL ATLAS는 단일 HTML 파일로 동작하는 불교 콘텐츠 뷰어 겸 에디터다. 설치도 서버도 필요 없다. 파일 하나를 더블클릭하면 열리고, 수정하고, 그대로 누군가에게 보낸다.

이 글은 v0.8(구버전 체계)에서 SemVer `0.0.1`로 새로 시작한 뒤, v0.0.6까지 6개 버전에 걸친 핵심 결정들을 정리한다.

## v0.0.1 — 정리: 듀얼 런타임 폐기

v0.8까지 ATLAS는 두 개의 런타임(편집용, 뷰어용)을 유지하고 있었다. 같은 코드를 두 벌로 관리하는 것은 비용이 지속적으로 증가하는 구조였다.

v0.0.1의 핵심은 **듀얼 런타임을 폐기하고 단일 런타임으로 통일**한 것이다. 이 결정으로 코드베이스가 단순해졌고, 이후 모든 기능 추가의 기반이 되었다.

함께 추가한 기능 두 가지:
- 문서뷰에 읽기 모드 토글 (편집 UI를 숨기고 본문만 표시)
- 사이드바 태그 영역에 "모든 태그 선택 해제" 버튼

## v0.0.2 — 편집자 시스템: 누가 무엇을 고쳤는가

단일 HTML 파일의 가장 독특한 특성은 **파일 자체가 편집기이면서 콘텐츠**라는 것이다. 누군가에게 파일을 보내면, 그 사람이 내용을 수정하고 다시 보낼 수 있다. 그러면 "누가 무엇을 고쳤는지"를 추적해야 한다.

v0.0.2에서 구현한 것:
- **편집자 등록 시스템**: OL 파일 저장 시 편집자 이름과 이메일 입력. 기존 편집자면 기록 누적, 새 편집자면 새로 등록
- **카드 행위 기록**: 각 카드(콘텐츠 단위)에 생성·수정·삭제 이력을 `meta.actLog`에 저장
- **히스토리 뷰**: About 탭에서 편집 기록을 시간순으로 확인

## v0.0.3 — 편집 기록의 영속성

v0.0.2에서 드러난 문제: 카드를 영구 삭제하면 해당 카드의 편집 기록도 함께 사라졌다. `_collectActsForEditor()`가 `cards`와 `trash`만 순회했기 때문이다.

v0.0.3에서 해결한 것:
- 영구 삭제 시 해당 카드의 `actLog`를 아카이브에 보존
- "비움"(프로젝트 창시자) 자동 등록 — 첫 편집자가 누락되는 문제 수정
- 히스토리 UI를 2컬럼 레이아웃으로 개선

## v0.0.4 — Dev 서버: 개발 환경 구축

여기까지 개발은 `npm run build` → 브라우저에서 파일 열기 → 확인 → 수정 → 다시 빌드... 의 반복이었다. 단일 HTML 특성상 핫 리로드가 까다로웠다.

v0.0.4에서 esbuild 내장 서버를 도입했다:

```
npm run dev 실행 시:
  1. esbuild context 생성 (watch 모드)
  2. 내장 서버 포트 3000 시작
  3. 소스 변경 감지 → 자동 재빌드
  4. SSE(EventSource)로 브라우저에 신호 → 자동 새로고침
```

빌드 산출물도 분리했다: `npm run build`는 `ol-atlas_v0.0.4.html`(버전 포함), `npm run dev`는 `ol-atlas_dev.html`(고정명).

## v0.0.5 — 홈 랜딩 페이지: 책으로서의 정체성

v0.0.4까지의 ATLAS는 "도구"였다. 칸반 보드가 열리고, 카드를 만들고, 편집했다. 하지만 ATLAS 위에 BOOK 콘텐츠를 탑재하면 사용자가 처음 보는 것은 칸반 보드가 아니라 **책 표지**여야 한다.

v0.0.5의 핵심은 홈 랜딩 페이지 신설이다:

- `meta.bookInfo` 스키마 확장 — 책 제목, 부제, 저자, 역자, 출판일, 한 줄 소개 등
- About 파일 정보 탭에 책 메타데이터 입력 폼 신설
- 홈 랜딩을 Mainline Astro 템플릿 스타일로 전면 재설계
- 상단 floating pill형 네비바
- 테마 토글 3단계: 라이트 → 다크 → 독서(세피아)

세피아 모드는 장시간 독서에 적합한 따뜻한 톤의 배경이다. 불교 콘텐츠의 성격과도 맞는다.

## v0.0.6 — 독서뷰: 읽기를 위한 전용 공간

v0.0.5까지 "읽기"는 문서뷰의 읽기 모드 토글에 의존했다. 편집 UI를 숨기는 방식이라 구조적 한계가 있었다 — 편집 기능의 CSS/JS가 그대로 로드되고, 레이아웃도 편집에 최적화된 채였다.

v0.0.6에서 독서뷰(Reader View)를 **완전히 새로운 뷰**로 분리했다:

- 기존 `dv-read-mode` 시스템 전체 제거
- `.rv-*` 전용 네임스페이스로 독립 구현
- 사이드바: 배경 투명, 문서 목록만 표시, hover 시 나타나고 벗어나면 사라짐
- 목차: 본문 우측에 투명하게 표시, 같은 hover 동작
- 하단 페이지 이동 버튼 (이전/다음 카드)
- 글자 크기 조정 버튼 (+/-)
- 헤더: 스크롤 다운 시 사라지고 스크롤 업 시 나타남

독서뷰의 설계 원칙은 존재 선언문의 "깊은 체류, 조용한 탐색"과 정확히 일치한다. UI가 콘텐츠를 방해하지 않고, 필요할 때만 나타난다.

## 현재 상태와 다음 단계

v0.0.6 기준으로 ATLAS는 세 가지 뷰를 갖추었다:

| 뷰 | 용도 |
|----|------|
| 홈 랜딩 | 책 표지, 소개, 입장 |
| 칸반/편집 | 콘텐츠 생성·편집·관리 |
| 독서뷰 | 몰입 독서 |

빌드 산출물은 단일 HTML 파일 하나. 평균 크기는 콘텐츠 없는 기본 버전 기준 약 400KB. 의존성 없음, esbuild만으로 빌드.

6,996줄에서 시작한 코드가 기능 추가와 함께 성장했지만, "단일 파일, 네트워크 독립, 로컬 우선"이라는 원칙은 한 번도 흔들리지 않았다.

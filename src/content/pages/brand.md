## OL 브랜드 정의서

**문서 버전**: v1.4  
**최종 업데이트**: 2026-06-26  
**기준 프로젝트**: OL HOME (`biwoom.github.io`)  
**기준 배포 URL**: `https://biwoom.github.io/`

이 문서는 OL 프로젝트의 브랜드 정체성을 정의합니다. OL을 함께 만들어가는 모든 참여자가 같은 기준을 공유하기 위한 약속이며, 기술적·디자인적·콘텐츠적 결정의 기준점입니다.

## 1. 브랜드 개요

### 1.1 브랜드명

| 항목 | 값 |
|------|------|
| 한글 | 올 |
| 영문 | OL |
| 발음 | [올] / [ol] |

### 1.2 핵심 정의

OL은 불교 문헌, 이야기, 시각 자료, 작업 기록을 디지털 환경에 맞게 정리하고 공개하는 불교 콘텐츠 프로젝트입니다. 백과사전식 나열을 지양하고, 문맥과 개념이 유기적으로 연결된 살아있는 학습 환경을 지향합니다.

현재 OL HOME은 Astro 기반 정적 사이트로 운영되며, 다음 주소를 공식 입구로 사용합니다.

```txt
https://biwoom.github.io/
```

### 1.3 태그라인과 슬로건

**태그라인**

- 한국어: *실처럼 이어진 지혜*
- 영문: *Thread by thread, truth unfolds*

**슬로건**

- *지혜의 올을 짜다* (Weaving the Wisdom)

## 2. 이름의 의미 구조

"올"이라는 한 음절에는 세 개의 층위가 겹쳐 있습니다.

### 층위 1 — 올, 실의 올

섬세함, 디테일, 텍스트의 기본 단위.

경전(經典)의 `經`은 날실을 뜻합니다. 산스크리트어 `sūtra`는 실, 줄을 의미하며 부처의 말씀을 실처럼 꿰어 놓았다는 뜻입니다. 인연의 실로 모든 존재가 연결되어 있다는 연기법, 선업과 악업의 올이 모여 삶의 천을 이루는 업보 사상까지, 불교의 핵심 세계관이 이 한 글자에 담겨 있습니다.

```txt
날실(經) = 경전(經典)
씨실(緯) = 논서(論書)
올 하나하나 = 개별 개념, 문장, 단어
천 전체 = 법계의 총체
직조 행위 = 학습과 깨달음의 과정
```

### 층위 2 — 오르다

위로 향함. 상구보리, 곧 위로 깨달음을 구한다는 뜻과 닿아 있습니다. 깊이 파고들수록 높이 올라가는 학습의 역설을 담습니다.

### 층위 3 — 올, 모두

일체, 법계. 부분이면서 동시에 전체를 포괄합니다.

### 인드라망과의 일치

화엄경의 인드라망처럼 각 올은 다른 모든 올과 연결됩니다. 하나를 당기면 전체가 움직이고, 부분과 전체가 서로 스며듭니다. OL의 온톨로지 기반 지식 그래프가 지향하는 구조도 이와 같습니다.

## 3. 존재 이유

OL의 출발점은 기술이 아닙니다.

> "나는 왜 불교 콘텐츠를 만들고 싶은가?"

그 답은 법을 만났을 때의 기쁨, 괴로움의 감소, 자비와 이타심, 존재에 대한 통찰, 명상을 통한 고요함과 자유를 다른 사람들과 나누고 싶다는 마음입니다.

따라서 OL의 핵심은 정보 전달이 아니라 **고통 감소와 삶의 전환을 돕는 경험**입니다.

모든 결정의 기준이 되는 질문은 다음입니다.

> **사람들이 이 경험을 통해 정말 더 자유로워지는가?**

## 4. 브랜드 철학

### 4.1 핵심 가치

**연결(Connection)**  
올 하나하나가 모여 천을 이루듯, 고립된 텍스트들을 살아있는 연결망으로 엮습니다.

**깊이(Depth)**  
천의 표면을 보는 것이 아니라, 올 하나하나를 따라가며 깊이 탐구합니다.

**여정(Journey)**  
사용자가 자신만의 올을 선택해 따라가는 학습 여정을 제공합니다.

### 4.2 디자인 철학

**최소주의**  
덜어낼수록 더 많이 담는다는 태도를 따릅니다.

**유기성**  
정적 데이터베이스가 아닌, 콘텐츠가 서로 연결되는 구조를 지향합니다.

**개방성**  
로그인 장벽 없이 접근할 수 있어야 합니다. 지식은 열려 있어야 하며, 공개 가능한 결과물은 가능한 한 공개합니다.

### 4.3 문사수

OL은 단순 지식 관리 시스템이 아니라 문사수 환경에 가깝습니다.

- **문(聞)**: 불교 이야기를 읽고 듣는다.
- **사(思)**: 질문하고, 사유하고, 연결하고, 삶과 대조한다.
- **수(修)**: 명상하고, 실천하고, 삶 속에서 체화한다.

## 5. 브랜드 체계

OL은 하나의 앱이 아니라 역할이 분리된 체계입니다.

### 5.1 현재 OL HOME 제품 구조

| 제품 | 역할 | 현재 상태 |
|------|------|------|
| **OL HOME** | 포털 | `biwoom.github.io` 저장소에서 GitHub Pages 루트 사이트로 운영 |
| **OL TEXT** | 문헌 콘텐츠 | 번역, 주석, 연구 노트를 중심으로 한 문헌 라이브러리 |
| **OL STORY** | 서사 콘텐츠 | 불교 인물·설화·가르침을 이야기 형식으로 재구성 |
| **OL DESIGN** | 시각 콘텐츠 | 인포그래픽과 시각 자료 라이브러리 |
| **OL NET** | 탐색 메뉴 | 인물·장소·개념 이름카드를 중심으로 STORY, TEXT, DESIGN을 연결하는 탐색 레이어 |
| **OL BLOG** | 작업 기록 | 콘텐츠 개발 과정과 코드 수정 과정 기록 |
| **OL ENTITY** | 지식 그래프 | 인물, 장소, 개념 등의 관계형 기준 데이터이자 개별 상세 문서 |
| **OL ATLAS** | 도구/아카이브 | 단일 HTML 산출물을 `public/atlas/`에서 제공 |

`BOOK`과 `WORKS` 라우트는 호환 및 과거 구조 연결을 위해 남아 있으나, 신규 운영의 중심은 `TEXT`, `STORY`, `DESIGN`, `NET`, `BLOG`입니다. `ENTITY`는 상단 공개 메뉴가 아니라, NET과 각 콘텐츠에서 연결되는 개별 상세 문서 체계로 유지합니다.

현재 공개 홈페이지의 상단 구조는 다음 흐름을 기준으로 합니다.

```txt
HOME / TEXT / STORY / DESIGN / NET (+ BLOG)
```

여기서 `NET`은 기존 `ENTITY` 문서를 대체하는 메뉴가 아니라, 이름과 관계를 먼저 훑어본 뒤 다시 STORY, TEXT, DESIGN, ENTITY 상세로 이어지는 탐색 입구입니다.

### 5.2 현재 저장소 구조

```txt
biwoom.github.io
├── src/content/text       → 문헌 콘텐츠
├── src/content/story      → 이야기 콘텐츠
├── src/content/design     → 시각 콘텐츠
├── src/content/blog       → 작업 기록
├── src/content/entities   → 지식 엔티티
├── src/content/ontology   → 온톨로지 기준
├── src/content/pages      → 브랜드·운영 기준 문서
└── public/atlas           → ATLAS 정적 HTML 아카이브
```

### 5.3 기술 원칙

- **정적 배포**: OL HOME은 Astro 정적 사이트로 빌드하여 GitHub Pages에 배포합니다.
- **콘텐츠 원본 우선**: 사람이 직접 관리하는 원본은 `src/content/` 안에 둡니다.
- **자산 파생 원칙**: `public/generated/`는 빌드 전 동기화로 생성되는 파생물입니다.
- **검색 가능성**: Pagefind 인덱스를 통해 정적 사이트 안에서 검색을 제공합니다.
- **개방성**: 공개 가능한 자료와 개발 기록은 가능한 한 사이트와 저장소에 남깁니다.

## 6. 디자인 시스템

### 6.1 핵심 원칙

OL의 디자인은 사용자가 디자인 자체보다 콘텐츠에 집중하도록 돕는 것을 목표로 합니다.

```txt
모던        → 무채색 베이스 + 정확한 1px 보더
심플        → 장식 제거, 충분한 여백, 명확한 위계
세련        → 정밀한 간격, 일관된 radius
질리지 않음 → 개인 취향이 강한 컬러 사용 지양
```

### 6.2 색상

현재 OL HOME은 CSS 토큰 기반 라이트/다크 모드를 사용합니다.

| 용도 | 라이트 | 다크 |
|------|--------|------|
| 배경 | `#FFFFFF` 계열 | `#0A0A0A` 계열 |
| 본문 | neutral ink | light neutral ink |
| 보조 텍스트 | muted neutral | muted light neutral |
| 테두리 | neutral border | dark border |

정확한 값은 `src/styles/tokens.css`와 관련 CSS 파일을 기준으로 합니다.

### 6.3 타이포그래피

한국어 본문의 가독성을 우선합니다. 현재 사이트는 `Pretendard`를 기본 외부 폰트로 사용하고, `Noto Sans KR`, `Inter`는 CSS fallback stack에서 함께 동작합니다. 본문과 UI 라벨은 공통 타이포그래피 토큰을 기준으로 통일합니다.

### 6.4 간격과 Radius

간격은 4px 단위의 절제된 리듬을 기본으로 하며, 카드와 버튼의 radius는 작고 일관되게 유지합니다.

### 6.5 아이콘

현재 OL HOME은 주로 인라인 SVG 아이콘을 사용합니다. 단일 HTML 산출물이나 독립 자산에서는 외부 런타임 의존을 최소화합니다.

## 7. 브랜드 보이스

### 7.1 톤

- **명상적**: 서두르지 않고 깊이 생각하게 합니다.
- **초대적**: 강요하지 않고 탐험을 권유합니다.
- **지혜로운**: 가르치려 하기보다 통찰을 나눕니다.

### 7.2 언어 스타일

- **간결함**: 짧고 명확한 문장
- **시적임**: 은유와 비유의 절제된 사용
- **존중함**: 사용자를 학습자이자 탐험가로 대우

### 7.3 피해야 할 톤

교조적, 권위적, 과도하게 기술적, 상업적, 마케팅적인 톤은 피합니다.

## 8. OL이 하지 않는 것들

이 목록은 기술적 한계가 아니라 의지적 결정입니다.

- 사용자 간 공유/협업을 강제하는 기능
- 배지, 점수, 연속 사용일 같은 게이미피케이션
- 푸시 알림과 학습 리마인더
- 소셜 로그인과 계정 중심 구조
- 사용 통계 수집과 추적
- 추천 알고리즘 중심의 체류 유도
- 광고와 후원 콘텐츠
- 별점, 좋아요, 댓글 등 평가 중심 기능

OL은 사용자를 더 오래 붙잡아 두는 것보다, 사용자가 자신의 삶으로 더 잘 돌아가는 것을 중요하게 봅니다.

---

## 9. 대상 독자

OL은 우선 일반 대중을 바라봅니다.

- 불교를 처음 접하는 사람
- 삶의 의미를 찾고 있는 사람
- 마음의 고통을 줄이고 싶은 사람
- 수행에 관심은 있으나 어렵게 느끼는 사람
- 깊은 이야기를 원하지만 지루한 설명은 원하지 않는 사람

따라서 OL의 콘텐츠는 쉽고, 명료하며, 아름답고, 집중 가능하며, 반복해서 다시 찾아오고 싶어야 합니다.

## 10. OL 결정 카드

OL 개발 중 어떤 결정 앞에 섰을 때 다음 질문을 확인합니다.

> 이 결정이 다음을 만족하는가?

1. 깊은 체류를 돕는가, 빠른 소비를 부추기는가?
2. 사용자의 사유를 돕는가, 방해하는가?
3. AI가 청하지 않은 개입을 하게 만드는가?
4. 사용자의 기록을 사용자에게 속하게 두는가?
5. 가르침으로부터의 자유로 이끄는가, 가르침에의 집착을 강화하는가?
6. 단순함을 늘리는가, 복잡함을 늘리는가?
7. 사용자가 OL에 더 오래 머물게 만드는가, 자기 삶으로 더 잘 돌아가게 만드는가?

하나라도 답이 어긋난다면 다시 생각합니다.
